# img — ImageGen

| | |
|---|---|
| **Назначение** | Мультимодельная генерация изображений |
| **Стек** | Next.js + Prisma |
| **Порт** | **3334** (исходный был 3000) |
| **БД** | SQLite + файлы-картинки |
| **Авторизация** | нет |

## Куда положить код

Исходники Next.js-приложения (`package.json`, `prisma/schema.prisma`, и т.д.) —
в `apps/img/`. Сборка `npm ci && npm run build` выполняется в `Dockerfile`.

## Данные

Картинки и SQLite хранятся в томе `/data/img` (примонтирован в compose).
Стартует с чистого листа — перенос данных для img не выполняется.

## Запуск (см. supervisord.conf)

`npm run start -- -p 3334`

## Примечание

Конфликт порта 3000 с seo решён: img на 3334, seo на 3332. OCR-задачи
используют `tesseract`, установленный в образе.

## Встраивание в дашборд (iframe)

img открывается плиткой-iframe внутри дашборда. Next.js **по умолчанию не шлёт**
`X-Frame-Options`, поэтому встраивание работает «из коробки». Если в проекте
включён заголовок (через `headers()` в `next.config.js` или middleware) — уберите
для img правило с `X-Frame-Options: DENY/SAMEORIGIN` (или `frame-ancestors`),
иначе плитка покажет пустоту.
