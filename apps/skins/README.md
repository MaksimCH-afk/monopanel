# Brand Name — Website Template

Нейтральный front-end шаблон, собранный из готового дизайна. Сохранены вся
вёрстка, оформление (CSS), адаптивность и все брейкпоинты, анимации,
JS-логика и SVG-графика. Удалён бренд, тексты заменены на плейсхолдеры,
картинки — на заглушки. Файл готов к ребрендингу «как есть».

## Структура

```
index.html                 — главная (hero, лента провайдеров, каталог игр, FAQ)
bonuses/  deposits/  application/  log-in/  withdraw/   — контентные страницы
privacy-policy/ cookies-policy/ terms-conditions/ responsible-gaming/ — юридические
css/styles.css             — все стили, переменные темы и медиазапросы
css/brand.css              — ФИРМЕННЫЕ СТИЛИ: единая точка правки под бренд
css/brand.casinorex.css    — пример заполнения brand.css (бренд CasinoRex)
css/brand.fraga.css        — пример АВТО-генерации из скриншотов (бренд Fraga)
tools/extract_brand.py     — скрипт авто-определения цветов из скриншотов → brand.css
out/                       — результаты работы скрипта (отчёт + превью палитры)
Dockerfile / Dockerfile.tools / docker-compose.yml — запуск в Docker (см. DOCKER.md)
refs/                      — сюда класть референс-скриншоты для экстрактора
js/main.js                 — меню, drawer, модалка, табы каталога, переключатель языков, редирект
assets/img/flags/          — флаги для переключателя языков (SVG, оставлены)
assets/{fonts,icons,video} — пустые папки под ваши ассеты
favicon.*                  — заглушки фавиконок (замените на свои)
robots.txt  sitemap.xml  .htaccess
```

## Фирменные стили (тема бренда)

Под конкретный бренд шаблон перекрашивается **через один файл — `css/brand.css`**,
без правок вёрстки и `styles.css`. Он подключён после `styles.css` на каждой
странице и переопределяет семантические токены (цвета меню/кнопок/заголовков/
текста/пикеров/акцента/фона, шрифт, размеры шрифтов, форма CTA-кнопки).

* Полная инструкция и таблица «чек-лист → токен → что меняет» — в **`BRANDING.md`**.
* Не хотите подбирать цвета вручную — **`tools/extract_brand.py`** определит палитру
  из референс-скриншотов и сгенерирует `brand.css` сам (пример: `css/brand.fraga.css`).
* Готовый пример заполнения вручную — **`css/brand.casinorex.css`**.
* Базовые цвета палитры (`--gold`, `--green`, `--bg`…) остаются в `:root` внутри
  `styles.css`; `brand.css` поверх задаёт фирменные значения.

## Что заменить под себя

1. **Логотип.** Сейчас это инлайновый SVG-плейсхолдер «LOGO» (класс `.brand-logo`)
   в золотой гамме макета — в шапке, drawer, футере и модалке. Замените разметку
   `<svg class="brand-logo …">…</svg>` своим SVG/`<img>`. Размеры под каждый контекст
   уже заданы в конце `css/styles.css` (секция *LOGO PLACEHOLDER*).
2. **Цвета/тема.** Все цвета — в `:root` (`--gold`, `--green`, `--bg` и т.д.).
   Радиус кнопок переключается атрибутом `data-btn-style` на `<html>` (`pill`/`rounded`/`sharp`).
3. **Редирект.** В `js/main.js`: `PARTNER_OFFER_URL = 'https://money.com/'` — поставьте свой URL.
4. **Домены/SEO.** `hreflang` в `<head>`, `og:image`, `robots.txt`, `sitemap.xml` —
   замените `example.com` на свой домен; теги `<title>`/`description`/JSON-LD — на свои.
5. **Фавиконки** (`favicon.ico/.svg/-32x32.png`, `apple-touch-icon.png`) — это заглушки.
6. **Каталог игр** задаётся массивом `GAMES` в `js/main.js`. Обложки рисуются на CSS
   (`.cover--0…5`), картинки игр не нужны.

## Заметки

* Иконки, обложки игр, слот-машина и колесо фортуны — **CSS/SVG, не картинки**,
  поэтому переживают «удаление изображений» без потерь.
* Адаптивность: брейкпоинты `1080 / 880 / 620 / 400 px` + языковой свитчер — на месте.
* Единственная растровая бренд-картинка (логотип `.webp`) удалена и заменена на SVG.
