# BRANDING — как применять фирменные стили

Этот шаблон — нейтральная «болванка». Под конкретный бренд он **перекрашивается
через один файл** — `css/brand.css` — без правок вёрстки и базовых стилей.
Задача процесса: по «определению бренда» (список цветов + референс-скриншоты
сайта-источника) добиться максимально похожего вида через передачу **цветов,
шрифтов и стиля**.

## Файлы и зоны ответственности

| Файл | Что внутри | Правим под бренд? |
|------|------------|-------------------|
| `css/styles.css` | Вся структура и оформление, раскладка, адаптивность, брейкпоинты. Использует семантические токены `--c-*`, `--fs-*`, `--cta-*`. | ❌ нет (общий для всех брендов) |
| `css/brand.css` | **Единая точка кастомизации**: значения токенов бренда. Грузится после `styles.css`. | ✅ да |
| `css/brand.<name>.css` | Сохранённые варианты под конкретные бренды (напр. `brand.casinorex.css`). | ✅ да |

Подключение в `<head>` каждой страницы (порядок важен — `brand.css` после `styles.css`):

```html
<link href="css/styles.css" rel="stylesheet"/>
<link href="css/brand.css"  rel="stylesheet"/>
```

## Что приходит на вход («определение бренда»)

1. **Список цветов** — как в постановке задачи:
   `меню`, `кнопки (CTA)`, `заголовки`, `текст`, `пикеры в списках`, `акцентные цитаты`, `фон`.
2. **Референс-скриншоты** сайта-источника (для шрифта, размеров, формы CTA-кнопки).
3. Иногда — отдельный **референс CTA-кнопки** (скопировать форму/тень/цвет).

## Соответствие: чек-лист → токен → что меняет

### Цвета
| Пункт чек-листа | Токен в `brand.css` | На что влияет |
|-----------------|---------------------|---------------|
| Меню | `--c-menu` / `--c-menu-active` | Пункты верхнего меню (обычный / активный-наведённый) |
| Кнопки (CTA) | `--c-cta`, `--c-cta-hi`, `--c-cta-2`, `--c-cta-text` | Основная золотая кнопка `.btn--gold` (градиент + текст) |
| Заголовки | `--c-heading` | Все `H1–H4` |
| Текст | `--c-text` + `--muted` | Абзацы основного контента и вторичный текст |
| Пикеры в списках | `--c-list-marker` | Маркеры списков (`ol/ul`) в контенте |
| Акцентные цитаты | `--c-accent` | Eyebrow-метки, выделения `.accent`, акцентные элементы |
| Ссылки | `--c-link` / `--c-link-hover` | Инлайновые и футерные ссылки, e-mail, hint в модалке |
| Вторичная кнопка | `--c-secondary` / `--c-secondary-2` / `--c-secondary-text` | `.btn--green` (напр. «Register Now» в drawer), онлайн-пилюля |
| Акцентная палитра | `--gold` / `--gold-hi` / `--gold-deep` | Базовый акцент: eyebrow, заголовки таблиц, цифры, активные табы, hover-границы, символы в арте игр |
| Рамки и свечение | `--accent-rgb` / `--secondary-rgb` | RGB-триплеты: обводка карточек (`--line-gold`), `--glow`, радиальные градиенты герое/карточек |
| Фон | `--c-bg` + `--bg/--bg-elev/--surface/--surface-2` | Основной фон и «поверхности» (шапка, карточки) |

### Стили (CSS)
| Пункт | Токен | На что влияет |
|-------|-------|---------------|
| Шрифт | `--font` / `--font-display` | Контент / заголовки. Платный → подобрать ближайший бесплатный |
| Размер шрифта осн. контента | `--fs-content` | `font-size` тела страницы |
| Размер шрифта в меню | `--fs-menu` | Пункты меню |
| Размер H1–H2 | `--fs-h1` / `--fs-h2` | Заголовки |
| Размер H3 | `--fs-h3` | Подзаголовки в контенте |
| Скругление углов | `--radius` / `--radius-sm` / `--radius-lg` / `--btn-radius` / `--cta-radius` | Карточки / мелкие элементы и инпуты / крупные карточки / кнопки-табы / CTA |
| Копия CTA-кнопки | `--cta-radius`, `--cta-shadow` (+ цвета CTA выше) | Форма/тень основной кнопки под референс |

## Пошагово

1. **Скопируйте** `css/brand.css` → `css/brand.<brand>.css` (стоковый файл оставьте как образец).
2. **Цвета.** Перенесите hex из списка в блок «1. ЦВЕТА». Для CTA выведите блик
   `--c-cta-hi` (светлее) и `--c-cta-2` (темнее) из основного цвета. Для фона —
   сделайте `--bg-elev/--surface` на 1–2 тона светлее `--bg`.
3. **Шрифт.** Поставьте семейство в `--font`/`--font-display` и **обновите `<link>`
   Google Fonts** в `<head>`. Если фирменный шрифт платный — подберите ближайший
   аналог (можно уточнить у ChatGPT по скриншоту).
4. **Размеры.** Сверьте по референсу `--fs-content`, `--fs-menu`, `--fs-h1`, `--fs-h2`.
5. **CTA-кнопка.** По референс-картинке кнопки задайте `--cta-radius` (pill/скруглённая/острая)
   и `--cta-shadow` (или `none`), плюс цвета CTA.
6. **Подключите** свой файл: переименуйте в `brand.css` или поменяйте `href` в `<head>`.
7. **Проверьте** в браузере на всех брейкпоинтах: `1080 / 880 / 620 / 400 px`.

## Авто-определение цветов из скриншотов

Чтобы не подбирать hex вручную, есть скрипт **`tools/extract_brand.py`** — он
анализирует референс-скриншоты бренда, сам раскладывает цвета по ролям
(фон, поверхности, текст, вторичный текст, заголовки, CTA/кнопки, акцент,
маркеры, меню) и выдаёт готовый `brand.css` + отчёт + SVG/PNG-превью палитры.

```bash
python3 tools/extract_brand.py shot1.png shot2.png ... \
    --name Fraga \
    -o css/brand.fraga.css \
    --report out/report.md \
    --swatch out/palette.svg
```

Можно указать папку вместо файлов — возьмёт все картинки из неё.

**Как он определяет роли (эвристики):**
* **фон** — самый тёмный заметный цвет с большой площадью;
* **текст / заголовки / меню** — самый светлый малонасыщенный (нейтрализуется до чистого);
* **вторичный текст** — смесь текста и фона;
* **CTA/кнопки** — самый насыщенный цвет с заметной площадью (блик `--c-cta-hi`,
  тёмный край `--c-cta-2` и тень выводятся автоматически; цвет текста на кнопке —
  по контрасту);
* **акцент/маркеры/рамки/ссылки** — второй насыщенный цвет с другим тоном; если такого нет —
  светлый оттенок CTA (так шум от логотипов/обложек игр не попадает в тему).

**Проверка контраста (WCAG).** Скрипт считает контраст текст/фон, текст-на-CTA и
вторичный текст/фон, авто-подгоняет светлоту текста до AA (≥ 4.5) и выбирает цвет
текста на кнопке по контрасту. В отчёте — таблица с PASS/FAIL: если, например,
текст на фирменной кнопке не дотягивает до 4.5, скрипт это пометит (цвет бренда
молча не меняет — решение за вами).

**Режим `--enforce-contrast`.** Если пара «текст на CTA» ниже AA — скрипт доводит её
до порога и пишет коррекцию в отчёт. Сначала пробует довести **только цвет текста**
до чистого чёрного/белого (для любого сплошного цвета один из них даёт ≥ 4.58:1, то
есть фирменный цвет кнопки остаётся нетронутым); и лишь если бы и этого не хватило —
минимально подвигает сам CTA. В отчёт попадает строка вида
`--c-cta-text: #14151a → #000000 (4.38:1 → 5.05:1)`.

**Варианты акцента.** `--variants N` дополнительно сгенерирует
`brand.<name>.alt1.css`, `…alt2.css` — те же роли, но с акцентом по другим
заметным цветам из палитры (удобно, когда у бренда несколько ярких цветов).

**Советы для точного результата:**
* кормите скрипту **UI-скрины** (меню, шапка, кнопки, drawer) — на них фирменные
  цвета «плоские» и крупные; витрины с обложками игр/логотипами команд дают шум;
* скрипт всегда печатает **top-кандидаты в акцент** — если авто-выбор CTA/акцента
  не совпал с брендом, возьмите нужный hex из таблицы и впишите в `brand.css`
  (типографику — размеры/шрифт — скрипт оставляет из шаблона, их задайте вручную по референсу);
* готовый пример работы скрипта — `css/brand.fraga.css` (бренд Fraga: фон `#111113`,
  текст `#eeeced`, CTA-красный `#d74d4c`), отчёт и превью — в папке `out/`.

## Готовый пример

`css/brand.casinorex.css` — заполнен по определению CasinoRex
(меню `#FFFFFF`, кнопки `#ECCBA2`, заголовки `#F3D1A6`, текст `#868EB0`,
пикеры `#EECDA3`, фон `#1C1D27`). Чтобы применить — переименуйте его в `brand.css`
или переключите `<link>` на него.

> Скриншоты-референсы могут быть из старой версии источника — на расхождения
> с текущим макетом шаблона не ориентируемся, берём из них только
> цвета / шрифт / размеры / стиль кнопки.
