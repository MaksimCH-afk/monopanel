# HubNero · Генератор URL-анкоров

Веб-инструмент для SEO: принимает домены и частотки ключей, рассчитывает план
размещения ссылок по типам анкоров (в процентах согласно выбранной стратегии) и
выгружает готовый **Excel** в формате задачи на размещение.

## Возможности

- **Стратегии** — профиль анкоров: процент безанкора + роли ключей
  (`основной 1/2`, `добавочный 1/2…`). У каждой стратегии задаётся **вид
  безанкора** (безанкорный профиль). Базовые: «Обычная» (70/12/9/5/4),
  «Безопасная» (75/13/5/4/3) и «Крауд + сабмиты» (100% безанкор). Можно
  создавать свои с любым числом добавочных ролей; сумма весов проверяется (=100%).
- **Тип кампании определяется стратегией.** Стратегия с ролями даёт вкладку
  «Прогоны»; стратегия из 100% безанкора (без ролей) — вкладку «Крауд+сабмиты».
  Внутренние страницы выгружаются отдельной вкладкой.
- **Безанкорные профили** — сохранённые наборы форматов (как стратегии): голый
  URL (`{url}` → `https://site.com/`) и голый домен (`{domain}` → `site.com`) в
  нужной пропорции. 3 базовых профиля + создание своих, с живым примером.
  Профиль выбирается внутри стратегии.
- **Внутренние страницы** — общий справочник `тип страницы + язык → суффикс`
  (en/de/pl/tr/pt-br) и пер-проектная настройка (язык анкоров + `тип страницы →
  URL-путь`). Анкор = `{бренд} {суффикс}`, по одной строке на страницу.
- **Импорт проектов из Excel** — один файл, где **каждая вкладка = проект**
  (ключи + частотность + домен на самой вкладке). Колонки распознаются по
  синонимам в любом порядке, колонка KD игнорируется, домен находится
  автоматически (`http→https`), пустые и сводные/дашборд-листы пропускаются.
  Проекты создаются или обновляются по домену.
- **Язык статьи** — выпадающий список языков; опция «Не указывать в выгрузке»
  убирает колонку Article Language.
- **Перераспределение отсутствующих ролей** — по умолчанию освободившийся % уходит
  в безанкор; есть ручная корректировка через JSON.
- **Генерация** — выбор проектов, со встроенным выбором стратегии / объёма / языка
  прямо в таблице и разбивкой анкоров по процентам (грузится по требованию).
  Параметры выгрузки: **Sprint** (одно значение на файл), **SEO Specialist**,
  формат (раздельные Excel / ZIP), режим строк (одна строка = одна ссылка **или**
  группировка с колонкой Link Q-ty).
- **Умное распознавание анкоров** (необязательно, галка на «Генерации») —
  исключает ключи, похожие **по смыслу** на стоп-анкоры (раздел «Стоп-анкоры»).
  Сверка идёт двумя моделями OpenRouter параллельно (список делится пополам); без
  ключей — запасной режим по вхождению слов.
- **Быстрая выгрузка** готового проекта в Excel прямо со списка проектов;
  **массовые действия** (удаление, назначение стратегии) по выбранным проектам.
- **История** — каждая генерация сохраняется в БД (проект, стратегия, объём,
  формат, число ссылок).
- **Логи** — детальный журнал событий (импорт, загрузки, правки, генерации,
  ошибки) с фильтрами по уровню и категории.
- **Справка на каждой странице** и **виджет шуток про SEO** (OpenRouter с
  локальным фолбэком).
- **Детерминированность**: одинаковый вход → одинаковый выход; сумма ссылок на
  URL всегда равна заданному объёму.

## Технологии

FastAPI · SQLAlchemy + SQLite · Jinja2 · openpyxl · Docker.

## Запуск через Docker (рекомендуется)

```bash
docker compose up --build
```

Откройте **http://localhost:9999**. БД SQLite хранится в volume `anchor_data`
(`/data/app.db`) и переживает перезапуски. Новые колонки добавляются миграцией
при старте — данные не теряются.

### Ключи для шуток про SEO (необязательно)

Виджет работает и без ключа — на встроенном списке (~60 шуток, без повторов).
Чтобы шутки генерировались моделями OpenRouter, ключи добавляются **в интерфейсе**:
раздел «Логи» → блок «Ключи OpenRouter» внизу → «Добавить ключ 1 / 2» (берёт
значение из буфера обмена) и «Проверить ключи». Два слота используют разные
модели, и запросы **чередуются** между ними — это снижает таймауты и лимиты.

Альтернатива — переменная окружения `OPENROUTER_API_KEY` в `.env` (используется
как запасной ключ, если в интерфейсе ничего не задано):

```bash
cp .env.example .env
# впишите OPENROUTER_API_KEY=sk-or-...
```

`.env` в репозиторий не попадает (в `.gitignore`).

## Локальный запуск без Docker

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 9999
```

## Тесты и CI

```bash
pip install -r requirements-dev.txt
pytest -q
```

Покрывают логику генерации (роли по частотности, контроль суммы = объёму,
перераспределение, дробление безанкора), импорт проектов из Excel, экспорт Excel
(колонки, разворот/группировка, язык, URL Type), умный фильтр анкоров и сами
эндпоинты (создание/генерация/скачивание/массовые действия). Прогоняются
автоматически в GitHub Actions (`.github/workflows/ci.yml`) на каждый push.

## Быстрый сценарий использования

1. **Импорт проектов из Excel** (или создание проекта вручную на вкладке
   «Проекты»).
2. **Стратегии** — проверьте/создайте профиль анкоров и выберите его вид безанкора.
3. **Внутренние страницы** (по желанию) — задайте язык и пути для проекта.
4. **Проект** — по блокам: частотка → стратегия → объём → генерация.
5. **Генерация** — выберите проекты, задайте Sprint / SEO Specialist / формат /
   режим строк → скачайте Excel. Готовый проект можно выгрузить и одной кнопкой
   из списка проектов.

## Структура

```
app/
  main.py          # FastAPI: сборка приложения, подключение роутеров
  routers/         # роуты по разделам: projects, generation, strategies,
                   #   profiles, suffixes, history, logs, anchors
  generator.py     # чистая логика генерации — без I/O, тестируемая
  service.py       # мост ORM ↔ генератор (стратегии, профили, разбивки, сегменты)
  helpers.py       # общие помощники роутеров (прогресс, история, сопоставление)
  templating.py    # общий экземпляр Jinja2
  excel_export.py  # сборка .xlsx (write-only, разворот/группировка) и ZIP
  parsing.py       # парсинг и умный импорт проектов из CSV/Excel
  anchor_filter.py # умный фильтр анкоров (семантика через OpenRouter + фолбэк)
  jokes.py         # SEO-шутки (батч-кэш OpenRouter + фолбэк без повторов)
  appsettings.py   # ключи OpenRouter (2 слота) и их хранение в БД
  models.py        # ORM-модели (SQLite)
  database.py      # подключение к БД
  seed.py          # базовые стратегии/профили/суффиксы/стоп-анкоры + миграции
  logging_util.py  # запись событий в Логи
  templates/       # Jinja2-шаблоны дашборда
  static/          # CSS, favicon
tests/             # юнит- и эндпоинт-тесты
Dockerfile · docker-compose.yml · requirements.txt · requirements-dev.txt
.github/workflows/ci.yml  # автозапуск тестов
```

## Колонки выходного Excel

Одна строка = одна ссылка (либо группировка — см. ниже).

| Sprint | SEO Specialist | Project | Project Url | URL Type | Link Type | Anchor Type | Anchor | Keyword | Article Language |
|---|---|---|---|---|---|---|---|---|---|

- **Sprint** — одно значение на весь файл (задаётся при генерации).
- **SEO Specialist** — из выпадающего списка (по умолчанию Miles Nashwood).
- **Project** — голый домен (`site.com`).
- **Project Url** — страница размещения (главная для прогонов/крауда, внутренняя
  для внутренних страниц).
- **URL Type** — `Main Page` для главной, `Inner Page` для внутренних страниц.
- **Link Type**, **Anchor Type**, **Keyword** — пустые (заполняются вручную).
- **Anchor** — текст анкора (ключ для анкорных ролей; URL или домен для безанкора).
- **Article Language** — язык статьи; не выводится, если у проекта выбрано
  «Не указывать в выгрузке».

Шапка — чёрная заливка с белым жирным текстом.

**Группировка.** В режиме «Сгруппировать» один анкор = одна строка, а количество
выводится в ведущей колонке **Link Q-ty** (например, 459 в одной ячейке вместо
459 одинаковых строк).
