Почта demo / mock Cloudflare D1

Как это работает

Приём почты на любой адрес вашего домена (что-угодно@ваш-домен) без почтового сервера — на Cloudflare. Готовим один раз, дальше письма сами копятся в базе, а эта админка их показывает.

1
Письмо приходит на *@ваш-домен
▼
2
Cloudflare Email Routing (правило «catch-all») ловит любой адрес
▼
3
Email Worker разбирает письмо и пишет строку в базу D1
▼
4
Эта админка (:3338) читает D1 по REST API и показывает письма по ящикам

Важно: Cloudflare не хранит почту и не даёт «ящиков» — это только маршрутизатор. «Ящик acc37@…» — это просто выборка из базы по адресу получателя. Новый адрес заводить не нужно: начали слать на него — он сразу работает (catch-all принимает всё).

Настройка — по шагам

  1. Домен на Cloudflare. Домен должен обслуживаться DNS Cloudflare (зона добавлена в аккаунт).
  2. Создать базу D1. В папке apps/mail/worker/: 
    npm install
npx wrangler d1 create mail
    Команда напечатает database_id — это и есть «D1 Database ID». Впишите его в wrangler.toml (скопируйте из wrangler.toml.example).
  3. Применить схему и задеплоить Worker: 
    npx wrangler d1 execute mail --remote --file=./schema.sql
npx wrangler deploy
  4. Включить приём почты. Вручную: Cloudflare → Email → Email Routing → включить, подтвердить MX/TXT, во вкладке Routing rules включить Catch-all → Send to a Worker → mail-catcher.
    Или сразу на много доменов скриптом: 
    CF_API_TOKEN=xxxx node scripts/setup-domains.mjs \
  ваш-домен.com другой-домен.com
  5. Подключить админку. Откройте ⚙ Настройки и впишите три значения: Account ID, D1 Database ID, API Token (см. ниже, откуда их взять). После сохранения бейдж сменится с demo / mock на Cloudflare D1.

Откуда брать три значения

Account ID
Проще всего: в Настройках вставьте токен и нажмите «Подобрать по токену» — Account ID подставится сам.
Вручную: он виден прямо в адресной строке дашборда — dash.cloudflare.com/<account-id> — а также на странице Overview любого домена, блок «API» → Account ID справа (кнопка «Click to copy»).
D1 Database ID
Проще всего: кнопка «Подобрать по токену» в Настройках покажет список ваших баз D1 — выберите mail, и Database ID подставится сам (команда не нужна).
Иначе это UUID вашей базы (не имя «mail»): он печатается при создании базы npx wrangler d1 create mail и виден в дашборде Workers & Pages → D1 → mail → Database ID.
API Token
Токен, которым админка читает базу. Создать: My Profile → API Tokens → Create Token → Custom token. Права:
  • Account → D1 → Read — обязательно (чтение писем);
  • Account → D1 → Edit — только если хотите удалять письма или запускать очистку прямо из админки;
  • в Account Resources выберите свой аккаунт.
Токен хранится и используется только на сервере, в браузер возвращается лишь маскированным (abc••••1234).

Токен для скрипта настройки доменов — другой: ему нужны права Zone → Email Routing (Edit) и Zone → DNS (Edit). Токен для чтения писем (здесь, в Настройках) — только D1.

Полезно знать

  • Адреса заводить по одному не нужно — только catch-all (лимит индивидуальных правил ~200 на домен).
  • Локальная часть адреса — только латиница/цифры/дефисы (ASCII до @).
  • Cloudflare пропускает входящие только с валидным SPF или DKIM — совсем «грязные» отправители не дойдут.
  • Письма копятся в D1, пока их не чистить. Для temp-mail включите авто-очистку по Cron (MAIL_RETENTION_DAYS в wrangler.toml).
  • Без кредов админка работает в mock-режиме с демо-письмами — можно посмотреть интерфейс до всякой настройки.

Воркер для Cloudflare

Это код, который ловит письма и пишет их в базу D1. Залейте его в Cloudflare руками (без wrangler): скопируйте кнопкой ниже и вставьте в редактор воркера в дашборде.

Порядок (всё через дашборд Cloudflare)

  1. Создать базу D1. Workers & Pages → D1 → Create → имя mail. Откройте её вкладку Console, вставьте схему (кнопка «Копировать схему» ниже) и выполните.
  2. Создать воркер. Workers & Pages → Create → Worker → задайте имя (напр. mail-catcher) → Deploy → Edit code. Удалите шаблон, вставьте код (кнопка «Копировать воркер») → Deploy.
  3. Привязать базу. Воркер → Settings → Bindings → Add → D1 database: Variable name DB, база mail → Save.
  4. Направить почту. На уровне домена: Email → Email Routing → включить, подтвердить MX/TXT → Routing rules → Catch-all → Send to a Worker → ваш воркер. (Воркер — один на аккаунт; этот шаг повторяется для каждого домена.)
  5. Подключить админку. Вернитесь в ⚙ Настройки и нажмите «Подобрать по токену» — база mail появится в списке.

Без внешних зависимостей. Воркер разбирает письмо (тема, текст, HTML, кодировки base64/quoted-printable, UTF-8) встроенным мини-парсером — никаких import из npm.
