Почта demo / mock Cloudflare D1

Как это работает

Приём почты на любой адрес вашего домена (что-угодно@ваш-домен) без почтового сервера — на мощностях Cloudflare. Настройка выполняется один раз, после чего письма автоматически сохраняются в базу данных, а админка их отображает.

1
Письмо приходит на *@ваш-домен
▼
2
Cloudflare Email Routing (правило catch-all) перехватывает адрес
▼
3
Email Worker разбирает письмо и записывает строку в базу D1
▼
4
Админка (:3338) читает D1 по API и группирует письма по ящикам

Важно: Cloudflare не хранит почту и не создаёт реальных «ящиков». Любой адрес получателя (например, acc37@ваш-домен) — это просто строка в базе данных. Заводить адреса заранее не нужно: как только на адрес отправят письмо, он тут же отобразится в панели.

Что где лежит

  • Админка (:3338) — веб-интерфейс для чтения писем из D1. Работает локально/на сервере, ничего в сам Cloudflare не деплоит.
  • apps/mail/worker/ — код Email Worker и файл структуры schema.sql. Деплоится в Cloudflare для перехвата почты. Если не используется CLI, код и схему можно перенести в панель Cloudflare вручную (кнопка ⧉ Воркер).
  • scripts/setup-domains.mjs — скрипт для автоматического включения почты и настройки catch-all сразу для списка доменов через API.

Настройка по шагам

  1. Подготовка домена. Домен должен быть добавлен в Cloudflare и полностью обслуживаться его DNS-серверами (статус Active).
  2. Создание базы данных D1. Через панель управления (интерфейс 2026 года):
    1. В главном меню аккаунта: Build → Storage & databases.
    2. Нажмите синюю кнопку Create (или Create database).
    3. Введите имя базы mail и подтвердите создание.
    4. Откройте базу → вкладка Console → вставьте содержимое schema.sql → Execute.
    Альтернативный путь через CLI: 
    cd apps/mail/worker/
npm install
npx wrangler d1 create mail
# скопируйте database_id в wrangler.toml (см. wrangler.toml.example)
npx wrangler d1 execute mail --remote --file=./schema.sql
  3. Деплой и привязка (Binding) воркера. Задеплойте код из apps/mail/worker/ командой npx wrangler deploy или создайте воркер вручную (Compute → Workers & Pages), вставив код из панели ⧉ Воркер.
    Критически важно — привязать базу к воркеру: настройки воркера → Settings → Bindings → Add → D1 database:
    • Variable name: DB (строго заглавными);
    • D1 database: ваша база mail.
    Без этой привязки воркер будет выдавать ошибку при попытке сохранить любое письмо.
  4. Включение приёма почты (Email Routing). Ручная настройка в новом интерфейсе:
    1. Compute → Email Service → Email Routing.
    2. Нажмите + Onboard Domain и выберите ваш домен.
    3. Cloudflare предложит добавить MX и TXT/SPF записи — подтвердите автоматическое добавление и дождитесь смены статуса Syncing → Active.
    4. Вкладка Routing rules → блок Catch-all address → Edit.
    5. Измените Action с Drop на Send to a Worker.
    6. Выберите ваш воркер в списке → Save.
    Автоматическая настройка (для множества доменов): 
    CF_API_TOKEN=xxxx node scripts/setup-domains.mjs ваш-домен.com другой-домен.com
    Верифицировать Destination Addresses не нужно — почта уходит напрямую в воркер, а не пересылается на сторонний ящик.
  5. Подключение админки. Откройте ⚙ Настройки и заполните три параметра — интерфейс переключится из демо-режима (mock) в рабочий режим (Cloudflare D1).

Откуда брать значения для настроек

Account ID
Виден прямо в адресной строке браузера при входе в панель: dash.cloudflare.com/<account-id>/.... Также доступен на странице Overview любого домена в блоке API справа.
D1 Database ID
Уникальный UUID вашей базы (не само слово «mail»). Находится в панели: Storage & databases → D1 → клик по базе mail → строка Database ID.
API Token
Создаётся в панели: My Profile → API Tokens → Create Token → Custom token. Необходимые права (Permissions):
  • Account → D1 → Read — обязательно для чтения писем;
  • Account → D1 → Edit — необходимо, если планируется удалять письма через админку;
  • Account Resources: обязательно выберите ваш конкретный аккаунт.

Про автоподбор ID: базовый токен для чтения D1 не имеет прав на просмотр структуры всего аккаунта, поэтому Account ID в админке нужно вводить вручную (кнопка «Подобрать» затем сама найдёт базу по этому Account ID).
Токен для скрипта автоматической настройки доменов (setup-domains.mjs) — отдельный, ему требуются права Zone → Email Routing (Edit) и Zone → DNS (Edit).

Полезно знать

  • Лимиты: catch-all позволяет принимать почту на бесконечное число адресов. Индивидуальные правила маршрутизации в Cloudflare ограничены лимитом около 200 правил на домен.
  • Формат адресов: локальная часть (до знака @) должна состоять только из латиницы, цифр и дефисов (ASCII).
  • Фильтрация спама: Cloudflare автоматически отклоняет входящие письма от отправителей с невалидными или отсутствующими записями SPF/DKIM.
  • Очистка базы: письма накапливаются в D1 бессрочно. Чтобы база не переполнялась, настройте автоочистку по Cron, задав переменную MAIL_RETENTION_DAYS в файле wrangler.toml.
  • Бесплатный тариф: для приёма почты, обработки воркером и хранения в D1 достаточно полностью бесплатного тарифа (Free plan). Платный Workers Paid требуется только для отправки писем наружу.

Идентификатор аккаунта Cloudflare. Проще всего — нажать «Подобрать по токену» ниже. Вручную: он виден в адресной строке дашборда dash.cloudflare.com/<account-id>, а также на странице Overview любого домена (блок «API» справа).
UUID вашей базы (не имя «mail»). Выдаёт команда npx wrangler d1 create mail; позже виден в Workers & Pages → D1 → mail.
Cloudflare → My Profile → API Tokens → Create Token → Custom. Права: Account · D1 · Read (и Edit, если удаляете письма из админки). Чтобы кнопка «Подобрать» нашла и аккаунт — добавьте ещё Account · Account Settings · Read (без него Account ID впишите вручную). Хранится только на сервере.

Воркер для Cloudflare

Это код, который ловит письма и пишет их в базу D1. Залейте его в Cloudflare руками (без wrangler): скопируйте кнопкой ниже и вставьте в редактор воркера в дашборде.

Порядок (всё через дашборд Cloudflare)

  1. Создать базу D1. Workers & Pages → D1 → Create → имя mail. Откройте её вкладку Console, вставьте схему (кнопка «Копировать схему» ниже) и выполните.
  2. Создать воркер. Workers & Pages → Create → Worker → задайте имя (напр. mail-catcher) → Deploy → Edit code. Удалите шаблон, вставьте код (кнопка «Копировать воркер») → Deploy.
  3. Привязать базу. Воркер → Settings → Bindings → Add → D1 database: Variable name DB, база mail → Save.
  4. Направить почту. На уровне домена: Email → Email Routing → включить, подтвердить MX/TXT → Routing rules → Catch-all → Send to a Worker → ваш воркер. (Воркер — один на аккаунт; этот шаг повторяется для каждого домена.)
  5. Подключить админку. Вернитесь в ⚙ Настройки и нажмите «Подобрать по токену» — база mail появится в списке.
