# Единая панель (monopanel)

Объединение 6 локальных приложений в **один Docker-контейнер** с дашбордом-плитками.
Каждое приложение работает как есть и слушает свой порт; страница-дашборд (`:333`)
даёт единую точку входа — плитки ведут на порты приложений.

> Приложения **не переписываются**. Объединение — на уровне точки входа (один
> образ + supervisord), а не кода: стеки разнородны (PHP, FastAPI ×2, Flask ×2,
> Next.js ×2), слить их в один код нереально.

---

## Приложения и порты

| Софт | Назначение | Стек | Порт | Маршрут плитки |
|------|-----------|------|------|----------------|
| **дашборд** | страница-плитки | Caddy (статика) | **333** | — |
| **cf** | Cloudflare Panel (домены, SSL, DNS, токены) | PHP 8.1 + Apache | **3331** | `/` |
| **seo** | GSC / SEO Dashboard | Next.js + Flask | **3332** | `/` |
| **ank** | Генератор анкоров (HubNero) | FastAPI + uvicorn | **3333** | `/` |
| **img** | ImageGen (мультимодельная генерация) | Next.js + Prisma | **3334** | `/` |
| **arc** | Web Archive Checker | FastAPI async | **3335** | `/` |
| **skins** | Brandskins (темы брендов) | Flask + gunicorn | **3336** | `/admin` |

- Старых портов (1000/9999/8000/3000/8888) в новой сборке не остаётся.
- Конфликт порта 3000 (seo и img) решён переназначением: seo→3332, img→3334.
- Flask-бэкенд seo (`5001`) — служебный, наружу не публикуется.
- `333` — привилегированный порт (<1024); под Docker обычно ок. Если демон
  откажется привязать — запасной вариант `3330` (поменять в `.env` и `docker-compose.yml`).

## Структура

```
monopanel/
├── apps/
│   ├── cf/          PHP (Cloudflare Panel)        → :3331
│   ├── seo/         Next + Flask                  → :3332 (+5001 внутр.)
│   ├── ank/         FastAPI                       → :3333
│   ├── img/         Next + Prisma                 → :3334
│   ├── arc/         FastAPI                       → :3335
│   └── skins/       Flask + gunicorn              → :3336 (/admin)
├── dashboard/
│   ├── index.html   страница-плитки (:333)
│   └── Caddyfile    конфиг статики
├── docker/
│   ├── entrypoint.sh    подготовка данных + старт
│   └── supervisord.conf запуск 6 процессов + дашборда
├── data/            ПОСТОЯННЫЕ данные (в .gitignore!) — см. data/README.md
├── Dockerfile       сборка единого образа
├── docker-compose.yml
├── .gitignore       закрывает data/, *.db, credentials.txt, .env
├── .env.example
└── README.md
```

Каждая папка в `apps/` содержит `README.md` с описанием: куда положить исходный
код приложения, как оно запускается и что изменено относительно оригинала.

### Как образ устроен внутри

- База `php:8.1-apache`; поверх ставятся Node 22, Python 3, tesseract, Caddy.
- У каждого Python-приложения свой venv (`/venv/ank`, `/venv/arc`, `/venv/seo`,
  `/venv/skins`) — из-за конфликта версий Flask (seo 2.3 / skins 3.1).
- `supervisord` держит все процессы, перезапускает упавшие, логи по каждому
  отдельно (префикс `[program:*]`).
- Дашборд — статика на Caddy (`:333`); плитки берут хост динамически (работает
  и по `localhost`, и по IP).

## Запуск

```bash
cp .env.example .env       # по желанию — ключи API (OpenRouter/OpenAI/Auth0)
docker compose up --build  # первая сборка долгая: 3 рантайма + 2 Next-сборки
```

Открыть дашборд: **http://localhost:333**

Логи: `docker logs -f panel` (помечены `[program:...]`).

## Перенос данных (только cf и ank)

Сохраняются данные **cf** (домены, аккаунты, токены, группы) и **ank** (проекты,
стратегии, профили). Остальные приложения стартуют с чистого листа.
Подробный порядок забора и размещения БД — в [`data/README.md`](data/README.md).

> ⚠️ `cloudflare_panel.db` содержит API-токены Cloudflare. Папка `data/` —
> в `.gitignore`, в репозиторий не коммитится.

## Безопасность

- Секреты не коммитятся: `data/`, `*.db`, `credentials.txt`, `.env` закрыты
  в `.gitignore`.
- Если GitHub PAT когда-либо попадал в чат/историю — считать его
  скомпрометированным и **отозвать** (GitHub → Settings → Developer settings →
  Personal access tokens → Revoke), даже если scope ограничен одной репой.
- Общий логин не включён — каждое приложение открыто на своём порту. При
  необходимости добавляется слой Caddy с одним паролем перед портами.

## Честные оговорки

- **Первая сборка тяжёлая и долгая**, образ крупный — осознанный компромисс ради
  «одного контейнера» для single-user инструмента.
- Это **первый рабочий каркас**, не вылизанный билд. Большой мульти-рантайм образ
  обычно требует 1–2 точечных правок под конкретную машину (версии пакетов,
  npm-зависимости). Падение на шаге сборки — ожидаемо, чинится в
  `Dockerfile` / `docker/supervisord.conf`.
- **seo + Auth0**: для входа нужны домен/ключи в `.env`; без них фронт поднимется,
  но вход не сработает.
- **skins** хранит состояние (выбранные темы) в файлах — каталог
  `apps/skins/instance` примонтирован томом, чтобы переживать пересборку.
- **iframe-нюансы**: если приложение отдаёт `X-Frame-Options: DENY/SAMEORIGIN`,
  встраивание в iframe заблокируется. Здесь плитки открываются в отдельной
  вкладке; для вкладок-iframe заголовок переопределяется на стороне прокси.

## Следующие шаги

- Положить исходники приложений в соответствующие папки `apps/*` (см. их README).
- Пройти первую сборку и поправить, если что-то отвалится.
- При желании — общий логин (один пароль на все плитки) через слой Caddy.
- При желании — встроить приложения в iframe внутри дашборда (вкладки вместо ссылок).
